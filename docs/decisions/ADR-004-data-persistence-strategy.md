# ADR-004: 데이터 보존: 호스트 바인드 마운트 원칙

- **Status:** ✅ Accepted
- **Date:** 2026-04-06
- **Decider:** Gemini CLI & User

## Context (배경)

- 컨테이너 기반 서비스 운영 시, 컨테이너 삭제 또는 업데이트 과정에서 데이터가 유실될 위험이 있음.
- 데이터의 영속성(Persistence) 확보와 백업 용이성이 필수적임.

## Decision (결정)

- 데이터 저장이 필요한 모든 서비스는 호스트의 특정 경로에 데이터를 직접 귀속시키는 **바인드 마운트(Bind Mount)** 방식을 원칙으로 함.
- 익명 볼륨(Anonymous Volume) 사용을 금지하며, 명시적인 경로 설정을 통해 데이터 가시성을 확보함.

## Consequences (결과)

- **이득:** 컨테이너 생명주기와 무관한 데이터 안전성 확보 및 호스트 수준에서의 직접적인 백업 가능.
- **비용:** 호스트 디렉토리 권한 설정 및 경로 관리가 복잡해질 수 있음.
