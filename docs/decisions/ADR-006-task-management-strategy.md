# ADR-006: 태스크 관리: 단계별 도구 확장 전략

- **Status:** ✅ Accepted
- **Date:** 2026-04-08
- **Decider:** Gemini CLI & User

## Context (배경)

- 프로젝트 운영을 위한 단축 명령어 및 자동화 스크립트가 늘어남에 따라 통합적인 관리 체계가 필요함.
- 현재의 단순 쉘 스크립트 방식에서 향후 의존성 증가에 대비한 확장 로드맵이 요구됨.

## Decision (결정)

- 프로젝트 성숙도에 따라 관리 도구를 단계적으로 확장하는 **'에스컬레이션(Escalation) 전략'**을 채택함.
- **1단계(현재):** 운영체제별 전용 스크립트(`gen-pdf.ps1`, `gen-pdf.sh`) 사용.
- **2단계(중기):** 명령어 복잡도 증가 시 `pyproject.toml` 기반의 `poethepoet` 도입.
- **3단계(장기):** 복합 의존성 및 프론트엔드 통합 필요 시 `package.json` 활용.

## Consequences (결과)

- **이득:** 현재의 가벼운 환경을 유지하면서도 향후 확장성을 보장하는 유연한 태스크 관리 가능.
- **비용:** 도구 전환 시점마다 설정 마이그레이션 오버헤드 발생.
