# ADR-003: 소스 관리: 분석용 소스 경로 최적화

- **Status:** ✅ Accepted
- **Date:** 2026-04-01
- **Decider:** Gemini CLI & User

## Context (배경)

- 외부 강의 자료, 분석용 로그, 임시 텍스트 파일(`lecture/`)이 루트 디렉토리에 혼재되어 프로젝트의 본질적인 구조 파악을 저해함.
- 하지만 해당 자료들은 프로젝트의 근거 데이터로서 형상 관리(Git) 대상에는 포함되어야 함.

## Decision (결정)

- 모든 비정형 분석 자료를 **`docs/lecture/`** 디렉토리로 통합 관리함.
- **웹 가시성 제어:** MkDocs 설정(`exclude_docs`)을 통해 웹 포털의 사이드바에서는 숨기되, 레포지토리 내부의 데이터 무결성은 유지함.

## Consequences (결과)

- **이득:**
  - 프로젝트 루트가 설정 파일(IaC, Config) 중심으로 정결해짐.
  - '문서화된 지식'과 '원천 데이터(Raw Data)'의 계층적 분리 달성.
- **비용:** 다른 문서에서 해당 경로를 참조할 때 상대 경로가 길어지는 불편함이 있음.
