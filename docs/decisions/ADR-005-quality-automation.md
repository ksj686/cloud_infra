# ADR-005: 품질 관리: pre-commit 프레임워크 전면 도입

- **Status:** ✅ Accepted
- **Date:** 2026-04-07
- **Decider:** Gemini CLI & User

## Context (배경)

- 다수의 스크립트(Bash, Python)와 YAML 설정 파일이 존재하는 인프라 프로젝트 특성상, 문법 오류나 시크릿 유출은 즉각적인 시스템 장애나 보안 사고로 이어질 수 있음.
- 형상 관리 시스템에 오염된 코드가 유입되기 전에 자동으로 검증하는 게이트웨이가 필요함.

## Alternatives Considered (대안 분석)

1. **CI/CD Pipeline (GitHub Actions 등) 전용 검사:** 중앙 집중형 검사가 가능하나, 개발자가 로컬에서 커밋을 완료한 후 원격지에 올렸을 때만 오류를 발견할 수 있어 피드백 루프가 늦음.
2. **Manual Formatting:** 휴먼 에러의 가능성이 매우 높고, 일관성 유지가 불가능함.

## Decision (결정)

- **`pre-commit`** 프레임워크를 프로젝트 표준으로 채택함.
- **핵심 훅 구성:**
  - **Gitleaks:** API 키 및 비밀번호가 코드에 포함되는 것을 사전에 차단.
  - **ShellCheck:** 쉘 스크립트의 잠재적인 버그 및 문법 위반 사항을 정적 분석.
  - **Prettier:** 마크다운 및 JSON/YAML 파일의 포맷을 강제하여 일관된 가독성 확보.

## Consequences (결과)

- **이득:**
  - 휴먼 에러 기반의 보안 사고 및 시스템 오설정 사전 예방.
  - 별도의 리뷰 없이도 팀 전반의 코드 품질 상향 평준화.
- **비용:** 초기 환경 설정의 복잡성과 커밋 시마다 수 초의 대기 시간이 발생함.
