# 클라우드 인프라 공통 표준 요소 (Core Common Elements)

본 문서는 프로젝트의 목적(보안, 데이터, 확장성 등)과 관계없이 모든 클라우드 인프라 구축 시 반드시 포함되어야 하는 공통 기술 표준 및 관리 원칙을 정의함.

---

## 1. 신원 및 권한 관리 (IAM: Identity and Access Management)

인프라의 모든 자원에 대한 접근 제어의 근간.

- **최소 권한 원칙 (Least Privilege):** 사용자나 서비스에 업무 수행에 필요한 최소한의 권한만 부여.
- **역할 기반 제어 (RBAC):** 개별 사용자가 아닌 역할(Role) 단위로 권한을 묶어 관리하여 일관성 유지.
- **다중 인증 (MFA):** 관리자 계정 및 외부 접근 접점에 대한 2단계 인증 필수 적용.
- **서비스 계정 분리:** 휴먼 계정(User)과 시스템 계정(Service Account)을 분리하여 감사(Audit) 추적성 확보.

## 2. 네트워크 기초 및 보안 (Network Fundamentals)

자원 간 통신 및 외부 노출을 제어하는 인프라의 혈관.

- **서브넷 격리 (Segmentation):** Public(외부 노출), Private(내부 전용), Data(DB 전용) 계층의 논리적 격리.
- **방화벽 기본 정책 (Default Deny):** 모든 인바운드 트래픽을 차단하고 필요한 포트만 화이트리스트 기반으로 개방.
- **암호화 통신 (Encryption in Transit):** 모든 트래픽 접점에 TLS(HTTPS) 적용 및 내부 통신 시 가급적 mTLS 또는 VPN 활용.
- **중앙 집중식 게이트웨이:** 리버스 프록시(Nginx, Traefik) 또는 로드 밸런서를 통해 외부 인입점 단일화.

## 3. 지능형 관측성 (Observability)

시스템 상태를 실시간으로 파악하고 장애를 예측하기 위한 필수 요소.

- **통합 로깅 (Centralized Logging):** 모든 서버와 서비스의 로그를 한곳(ELK, Loki 등)으로 수집하여 상관관계 분석 수행.
- **메트릭 모니터링:** CPU, 메모리, 디스크 등 하드웨어 자원과 서비스 응답 시간(Latency) 등의 성능 지표 실시간 수집.
- **임계치 알림 (Alerting):** 정상 범위를 벗어난 지표 감지 시 즉각적인 담당자 통보 체계 구축.
- **감사 추적 (Audit Log):** 시스템 설정 변경 및 주요 명령어 실행 이력을 보존하여 사후 사고 분석 지원.

## 4. 코드 기반 자동화 (Automation as Code)

인적 실수를 배제하고 동일한 환경을 즉시 복제하기 위한 엔지니어링 표준.

- **인프라 프로비저닝 (IaC):** Terraform, CloudFormation 등을 통해 인프라 자원의 생성과 변경을 코드로 관리.
- **구성 관리 (CM):** Ansible, Chef, Puppet 등을 활용하여 OS 설정 및 패키지 설치 상태를 일관되게 유지(Idempotency).
- **버전 관리 (Git):** 모든 인프라 관련 코드와 설정 파일을 Git 리포지토리에서 관리하여 변경 이력 추적.
- **파이프라인 연동:** 코드 변경 시 자동 검증 및 배포를 위한 CI/CD 체계 구축.

## 5. 보안 하드닝 및 취약점 관리 (Hardening)

공격 표면(Attack Surface)을 최소화하기 위한 상시 보안 활동.

- **OS 하드닝:** 불필요한 서비스/데몬 제거, 커널 파라미터 최적화, 보안 패치 자동화.
- **비밀값 관리 (Secrets Management):** API 키, 패스워드 등을 코드에 노출하지 않고 전용 보관소(Vault, Cloud Secrets)에서 관리.
- **취약점 스캔:** 정기적인 OS 패키지 및 애플리케이션 라이브러리에 대한 알려진 취약점(CVE) 스캔 및 조치.

## 6. 데이터 보호 및 복구 (Persistence & DR)

비즈니스 연속성(BCP)을 보장하기 위한 최후의 보루.

- **증분 백업 및 중복 제거:** 백업 효율성을 높이기 위한 기술적 최적화 적용.
- **RPO/RTO 정의:** 복구 시점 목표(RPO)와 복구 시간 목표(RTO)를 수립하고 이를 준수하기 위한 기술 구현.
- **복구 테스트:** 백업 데이터의 무결성을 확인하기 위해 정기적인 복구 시뮬레이션 수행.
- **오프사이트 보관 (Off-site Backup):** 원본 데이터와 물리적으로 분리된 장소(또는 별도 리전/클라우드)에 백업본 보관.

---

## 결론: 공통 요소의 중요성

- 위 요소들은 인프라의 **'위생 요인(Hygiene Factors)'**으로, 하나라도 누락될 경우 상위 서비스의 안정성이 근본적으로 흔들릴 수 있음.
- 본 프로젝트의 **Phase 1~6**은 이러한 공통 표준을 순차적으로 구현하여 **'가장 완벽한 인프라 템플릿'**을 완성하는 과정임.
