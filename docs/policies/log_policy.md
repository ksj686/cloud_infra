# Policy: 로그 보존 및 관리 정책 (Log Management)

## 📌 개요
- 시스템의 투명한 기록 유지 및 보안 사고 분석 근거 마련함.
- 디스크 점유 최적화 및 법적 로그 보존 요건(ISMS 등) 준수 지향함.

## 🗂️ 1. 로그 분류 및 중요도 (Log Classification)
- **보안 감사 로그:** `auditd` 이력, SSH 로그인 성공/실패, 권한 상승(`sudo`).
- **시스템 상태 로그:** `syslog`, `auth.log`, 커널 메시지(`dmesg`).
- **서비스 운영 로그:** Nginx Access/Error log, 애플리케이션 로그.

## 📅 2. 로그 순환 및 보존 (Rotation & Retention)
- **순환 주기 (Rotation):**
    - **일 단위:** 트래픽이 많은 웹 로그, 보안 로그.
    - **주 단위:** 일반 시스템 로그.
- **보관 기간 (Retention Policy):**
    - **활성 보관 (Uncompressed):** 서버 로컬에 최근 14일분 보존함.
    - **장기 보관 (Compressed):** 별도 로그 서버 또는 아카이브에 최소 90일~1년 보관함.
    - **용량 기반:** 단일 로그 파일이 100MB 초과 시 주기와 상관없이 즉시 순환함.

## 🛠️ 3. 로그 처리 및 최적화 (Processing)
- **압축:** 아카이브 시 `gzip`을 활용하여 저장 공간 70% 이상 절감함.
- **삭제:** 보관 주기가 지난 로그는 매일 새벽 크론탭(`find`)을 통해 자동 영구 삭제함.
- **모니터링:** 비정상적 로그 발생 시(에러 급증) 실시간 Slack 알림 연동함.

## ✅ 4. 로그 무결성 및 보안 (Security)
- **접근 권한:** 로그 디렉토리(`/var/log`) 및 파일 권한을 `640` 이하로 제한함.
- **수정 방지:** `auditd`를 통해 주요 로그 파일에 대한 변조 및 삭제 행위 실시간 감시함.
- **원격 전송:** 서비스 무중단 보장을 위해 중앙 집중형 로그 관리 서버(SIEM/ELK 등)로 실시간 전송 검토함.
