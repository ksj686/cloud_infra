# Docker 관리 및 운영 전략 가이드

## Docker 도입 효과
- **격리성:** 컨테이너 단위 독립 실행 환경 제공.
- **경량화:** 호스트 커널 공유를 통한 자원 효율성 극대화.
- **일관성:** 이미지 기반 배포로 개발-운영 환경 격차 해소.

## 핵심 개념
- **이미지(Images):** 컨테이너 생성을 위한 읽기 전용 템플릿.
- **컨테이너(Containers):** 이미지의 격리된 실행 인스턴스.
- **레이어(Layers):** 이미지 빌드 단위로 데이터 공유 및 용량 최적화 지원.

## 실무 운영 모범 사례
1. **불변성 유지:**
   - **지양:** 실행 중인 컨테이너 직접 수정(데이터 휘발 위험).
   - **지향:** `Dockerfile` 수정 후 이미지 재빌드(재현성/보안 확보).
2. **데이터 영속성 확보 방식 (Persistence Strategies):**
   - **방식 1: 바인드 마운트 (Bind Mount)**
     - 정의: 호스트의 실제 절대 경로와 컨테이너 경로를 직접 연결 (`-v /host/path:/container/path`)
     - 용도: 호스트 설정 파일(`nginx.conf`) 동기화, 실시간 소스 코드 반영, 인프라 관리자 직접 백업 필요 시
   - **방식 2: 네임드 볼륨 (Named Volume)**
     - 정의: 도커가 관리하는 별도 영역 내에 이름을 가진 저장소 생성 (`-v volume_name:/container/path`)
     - 용도: 컨테이너 간 데이터 공유, 도커 중심의 깔끔한 데이터 관리, 호스트 경로 의존성 제거
   - **방식 3: 익명 볼륨 (Anonymous Volume)**
     - 정의: 이름 없이 해시값으로 생성되는 일회성 저장소 (`-v /container/path`)
     - 용도: 성능 향상을 위한 임시 데이터 처리, 컨테이너 삭제 시 함께 삭제되어도 무방한 휘발성 데이터
   - **데이터 성격별 권장 전략:** 
     - **DB 데이터:** 데이터 격리 및 도커 통합 관리를 위해 **네임드 볼륨** 사용 권장
     - **설정/로그:** 호스트에서의 즉각적인 수정 및 가시성 확보를 위해 **바인드 마운트** 사용 권장

3. **데이터 성격별 마운트 활용 사례:**
   - **로그(Logs):** 서비스 운영 로그 보존 및 외부 로그 수집기(Loki 등) 연동을 위한 `/var/log` 경로 귀속
   - **설정(Configs):** 컨테이너 재빌드 없는 환경 설정 변경을 위해 `nginx.conf`, `prometheus.yml` 등 마운트
   - **업로드 파일(Uploads):** 사용자 생성 미디어 및 정적 자산의 영구적 보관
   - **보안 자산(Certs):** SSL 인증서 및 SSH 키의 중앙 집중 관리 및 컨테이너 간 공유
   - **모니터링(Sockets):** 호스트 상태 감시를 위한 `/var/run/docker.sock` 및 시스템 디렉토리(` /proc`, `/sys`) 접근

4. **이미지 최적화:** `alpine`, `distroless` 등 경량/보안 베이스 이미지 사용.
5. **보안 강화:** **Trivy** 등 도구 활용 취약점 상시 스캔.
   - 명령어: `trivy image [이미지명]:[태그]`

## 주요 명령어
- **이미지 빌드:** `docker build -t [이미지명]:[태그] .`
- **컨테이너 실행:** `docker run -d -p [호스트포트]:[컨테이너포트] [이미지명]`
- **상태 확인:** `docker ps` (실행 중), `docker ps -a` (전체)
- **리소스 정리:** `docker rm -f [컨테이너ID]`, `docker system prune -a` (미사용 데이터 일괄 제거)
