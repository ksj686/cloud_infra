# 심볼릭 링크(Symlink) 기반 버전 관리 가이드

## 도입 목적

- **무중단 업데이트(Zero-downtime):** 서비스 중단 없이 신규 버전 교체 가능.
- **추상화:** 고정 경로(예: `/usr/local/bin/app`) 활용으로 환경 설정 일관성 유지.

## 주요 활용 사례

- **DB 엔진:** 바이너리 경로 고정 및 버전 간 신속 전환.
- **웹 앱:** 신규 빌드 배포 후 링크 업데이트로 즉시 적용.
- **라이브러리:** 공유 라이브러리(`.so`) 호환성 유지 및 업데이트.

## 기본 명령어

- **링크 생성:** `ln -s [대상경로] [링크이름]`
- **링크 강제 업데이트:** `ln -sf [신규경로] [링크이름]`
- **대상 확인:** `ls -l [링크이름]`

## 실무 운영 전략

1. **버전별 격리:** 독립된 디렉토리에 신규 버전 사전 준비.
2. **원자적 업데이트:** `ln -sf` 활용 단일 명령으로 서비스 전환.
3. **신속 롤백:** 장애 발생 시 이전 버전 디렉토리로 링크 즉시 복구.
