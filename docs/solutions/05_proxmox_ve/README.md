# Proxmox VE (PVE) Starter Guide

Proxmox VE(Virtual Environment)는 기업용 오픈소스 가상화 관리 플랫폼으로, 강력한 성능과 유연한 관리 UI를 제공하는 Type-1 하이퍼바이저입니다.

### 1. 핵심 개념 (Core Concepts)

- **Type-1 하이퍼바이저**: Debian Linux 기반의 베어메탈 설치형 가상화 솔루션.
- **오픈소스**: KVM(전체 가상화)과 LXC(컨테이너 가상화)를 동시에 지원.
- **웹 기반 UI**: 별도의 클라이언트 설치 없이 브라우저에서 모든 클러스터 및 자원 관리 가능.
- **Community Edition**: 무료로 모든 핵심 기능을 사용 가능하며, 안정적인 엔터프라이즈 리포지토리 구독 시 유료 지원 제공.

### 2. 설치 단계 (Installation Steps)

1. **ISO 다운로드**: Proxmox 공식 홈페이지에서 최신 PVE ISO 이미지 다운로드.
2. **부팅 미디어 제작**: Rufus 또는 BalenaEtcher를 사용하여 USB 부팅 미디어 생성.
3. **설치 프로세스**:
   - 대상 디스크 선택 및 파일시스템(ZFS 등) 설정.
   - 네트워크 설정(Static IP 필수, FQDN 호스트네임 지정).
   - 관리자(root) 암호 및 이메일 주소 입력.
4. **관리 UI 접속**: 설치 완료 후 출력되는 `https://[서버IP]:8006` 주소로 접속.

### 3. 주요 스토리지 유형 (Storage Types)

- **ZFS**:
  - 소프트웨어 RAID 기능을 내장하여 고가의 RAID 컨트롤러 없이 데이터 보호 가능.
  - 강력한 스냅샷, 압축, 중복 제거 기능 제공.
  - 하드웨어 안정성을 위해 고용량 RAM 권장 (ARC 캐시).
- **LVM-thin**:
  - 오버프로비저닝을 지원하는 씬 프로비저닝 방식.
  - 실제 사용한 용량만큼만 물리 디스크 할당하여 공간 효율 극대화.
  - 로컬 스토리지에서 빠른 성능을 제공하지만 클러스터 간 공유에는 제한적.

### 4. LXC 컨테이너 vs VM (Full Virtualization) 가이드

| 구분          | LXC (Linux Container)                      | VM (Virtual Machine)                       |
| :------------ | :----------------------------------------- | :----------------------------------------- |
| **커널 공유** | 호스트 커널 공유 (오버헤드 매우 낮음)      | 독립적인 커널 사용 (완전한 격리)           |
| **용도**      | 경량화 서비스 (Nginx, Database, Docker 등) | 타 OS(Windows), 하드웨어 패스스루 필요 시  |
| **리소스**    | CPU/RAM 할당 유연함 (동적 할당)            | 예약된 고정 리소스 할당                    |
| **추천**      | 빠른 배포와 자원 절약이 우선인 경우        | 강력한 보안 및 호스트 독립성이 중요한 경우 |
