# PVE 고가용성 클러스터 (HA Cluster) 구축

Proxmox VE의 핵심 기능 중 하나인 HA(High Availability)는 노드 장애 발생 시 서비스를 자동으로 다른 노드로 이전하여 다운타임을 최소화합니다.

### 1. HA 클러스터 요구 사항 (Requirements)

- **노드 수**: 최소 3개 노드 권장 (쿼럼 유지를 위해 홀수 개수 권장).
- **네트워크**: 클러스터 통신(Corosync) 전용 저지연 네트워크 망 구축 권장.
- **공유 스토리지 (Shared Storage)**:
  - VM/LXC 데이터를 모든 노드에서 접근 가능해야 함 (NFS, iSCSI, Ceph 등).
  - 공유 스토리지 부재 시, ZFS 복제(Replication) 기능을 활용한 유사 HA 구현 가능.

### 2. 쿼럼(Quorum) 개념 이해

- **정의**: 클러스터의 의사 결정을 위해 필요한 최소한의 투표 수.
- **중요성**: 네트워크 분할(Split-brain) 현상 방지를 위해 클러스터 내 과반수 이상의 노드가 활성 상태여야 함.
- **투표 가중치**: 기본적으로 각 노드는 1표를 가지며, 2개 노드 클러스터에서는 한 노드만 죽어도 쿼럼이 상실되므로 QDevice(외부 타이브레이커) 사용이 필수적임.

### 3. SPoF(단일 장애점) 테스트 시나리오

클러스터의 안정성을 검증하기 위해 다음과 같은 테스트를 수행합니다.

1. **HA 리소스 등록**:
   - 관리 UI에서 특정 VM을 HA 그룹에 추가하고 'Started' 상태로 설정.
2. **노드 장애 시뮬레이션**:
   - VM이 가동 중인 노드의 전원 플러그를 뽑거나 강제로 시스템 종료(`poweroff`).
3. **자동 복구 확인**:
   - 클러스터가 해당 노드의 'Fence'(분리)를 감지하는지 모니터링.
   - 쿼럼이 유지되는 경우, 해당 VM이 생존 노드 중 하나로 자동 마이그레이션되는지 확인.
   - 서비스(VM 내 웹서버 등)의 가용성 재개 여부 및 소요 시간 체크.
4. **Fencing(펜싱) 메커니즘**:
   - 장애 노드가 좀비 상태로 살아나 데이터를 오염시키는 것을 방지하기 위해 강제로 격리/재부팅하는 과정.
   - 하드웨어 수준의 Watchdog 또는 IPMI를 활용한 펜싱 설정 권장.
