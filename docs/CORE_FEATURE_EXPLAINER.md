# Core Engineering Features & Security Strengths

본 문서는 프로젝트의 기술적 차별성과 보안 무결성을 보장하는 핵심 아키텍처 및 엔지니어링 전략 정리

---

## 1. 로컬 보안 가드레일 (Shift-Left Security)

코드 작성 및 커밋 단계에서 결함을 사전에 차단하는 강력한 로컬 방어 체계

- **Gitleaks 기반 시크릿 유출 원천 차단:** API Key, Secret Token 등 민감 정보의 리포지토리 업로드 시도 실시간 감지 및 거부
- **ShellCheck 기반 스크립트 품질 보장:** 인프라 실습 쉘 스크립트의 문법 오류, 논리적 버그, 잠재적 위험 요소를 커밋 전 정밀 진단
- **개인키 및 대용량 파일 탐지:** SSH/TLS 개인키 유출 방지 및 500KB 초과 대용량 파일 차단을 통한 리포지토리 최적화
- **Prettier 기반 스타일 표준화:** 참여자의 에디터 설정과 무관하게 모든 문서 및 설정 파일의 줄맞춤, 들여쓰기를 자동으로 교정하여 일관된 가독성 유지

---

## 2. 엔터프라이즈급 고가용성 설계 (High Availability)

단일 장애점(SPoF)을 배제하고 서비스 지속성을 극대화하는 아키텍처

- **MariaDB Galera Cluster:** 3-Node 기반 동기식 멀티 마스터 복제를 통해 데이터 유실 방지 및 즉각적인 장애 조치(Failover) 보장
- **ProxySQL DB 부하 분산:** 데이터베이스 레이어의 L7 프록시 운용을 통해 읽기/쓰기 분리 및 노드 상태 기반의 지능적 트래픽 배분
- **Proxmox 하이퍼바이저 HA:** 물리 노드 장애 시 가상 머신(VM)의 자동 재시작 및 이동을 통한 인프라 레벨의 가동률 확보
- **Ceph 분산 스토리지 연동:** 데이터 삼중화(Replication)를 통한 스토리지 하드웨어 결함 대비 및 무중단 용량 확장 체계 구축

---

## 3. 지능형 관측성 및 자동화 (Observability & Automation)

데이터 기반의 정밀 모니터링과 인프라 운영의 코드화

- **Thanos 기반 멀티 클러스터 통합:** 여러 환경의 프로메테우스 데이터를 통합 관리하고 객체 스토리지를 통한 영구적 로그 보관 지원
- **k6 정밀 부하 테스트:** 단순 평균 응답 시간이 아닌 p95(상위 5%) 지연 시간을 기준으로 인프라 성능 한계를 정량적으로 검증
- **AI 기반 선제적 자원 관리:** MS AI 예측 모델과 KEDA를 연동하여 부하 발생 전 선제적으로 자원을 증설하는 Proactive Scaling 구현
- **선언적 인프라 관리(IaC/GitOps):** Terraform 및 Ansible을 통한 환경 복제 자동화와 Argo CD 기반의 상태 동기화 체계 수립

---

## 4. 기대 효과 및 전문성 어필

- **운영 안정성:** 다중 보안 방어 및 가용성 설계를 통해 엔터프라이즈급 서비스 가동률 확보
- **협업 효율성:** 자동화된 품질 관리 도구 도입으로 지엽적 코드 리뷰 최소화 및 로직 집중 가능
- **확장 용이성:** 모든 구성 요소의 모듈화 및 코드화를 통해 하이브리드 클라우드로의 신속한 확장 기반 마련
