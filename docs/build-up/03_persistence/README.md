# [Phase 3] Persistence: 데이터 및 스토리지 가용성

데이터 손실 방지 및 고가용성 저장소 구축을 위한 기술 실행 절차

---

## 1. 가용 스토리지 구성 (RAID/LVM)

- **RAID 구축:** 하드웨어 장애 대비를 위한 RAID 1(미러링) 또는 RAID 5 구성
- **LVM 관리:** 가변적 파티션 설정을 위한 LVM 볼륨 그룹(VG) 및 논리 볼륨(LV) 생성
- **ZFS 검토:** 데이터 무결성 체크섬 및 스냅샷 기능을 위한 ZFS 미러링 도입 (선택 사항)

## 2. 분산 스토리지 연동 (Ceph)

- **Ceph 클러스터:** Proxmox 내 Ceph 활성화 및 네트워크 기반 데이터 삼중화(Replication) 설정
- **스토리지 풀 관리:** VM 및 컨테이너가 사용할 수 있는 공유 저장소(RBD) 제공

## 3. 컨테이너 데이터 보존 전략

- **네임드 볼륨 (Named Volume):** DB 데이터 등 영구 보존이 필요한 자산의 도커 통합 관리
- **바인드 마운트 (Bind Mount):** 설정 파일(`nginx.conf`) 및 로그 파일의 호스트 동기화

## 4. 엔터프라이즈 백업 (PBS)

- **Backup Server 구축:** 전용 백업 서버(Proxmox Backup Server) 설치 및 클러스터 연동
- **재해 복구(DR) 수립:**
  - **RPO:** 24시간 이내 데이터 복구 가능성 확보 (일 단위 백업)
  - **RTO:** 스냅샷 기반의 수 시간 이내 서비스 정상화 체계 검증
