# [Phase 6] Scalability: IaC 및 고가용성 인프라

인프라 프로비저닝 자동화 및 서비스 확장성 확보 절차

---

## 1. 인프라 프로비저닝 자동화 (IaC)

- **Terraform 도입:** Proxmox API 연동을 통한 가상 머신(VM) 및 네트워크 리소스의 선언적 관리
- **Ansible 구성 관리:** 모든 서버의 설정, 보안 하드닝, 패키지 설치 공정의 코드화(Playbook) 및 실행

## 2. DB 고가용성 클러스터 (MariaDB)

- **Galera Cluster 구축:** 3-Node 동기식 복제 클러스터 구성을 통한 데이터 무손실 보장
- **ProxySQL 설정:** 지능형 쿼리 라우팅 및 읽기/쓰기 분리를 통한 성능 최적화와 장애 조치 자동화

## 3. 유연한 인프라 확장 (Next Step)

- **오토 스케일링:** 부하 기반 리소스 동적 할당 및 인스턴스 자동 확장 체계 연구
- **하이브리드 연동:** 온프레미스와 퍼블릭 클라우드 간의 리소스 연동(Cloud Bursting) 및 GitOps 기반 상태 동기화
