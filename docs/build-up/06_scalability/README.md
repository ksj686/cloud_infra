# [Phase 6] Scalability: IaC 및 고가용성 인프라

인프라 프로비저닝 자동화 및 엔터프라이즈급 서비스 확장성 확보 절차 정리

---

## 1. 코드형 인프라 관리 (IaC)

인적 실수 배제 및 환경 복제 속도 극대화를 위한 자동화 공정

- **Terraform 프로비저닝:**
  - Proxmox API 연동을 통한 가상 머신(VM), 네트워크, 스토리지 리소스의 선언적 관리
  - **상태 관리:** 인프라 현재 상태와 코드 정의 상태의 실시간 동기화 체계 수립
- **Ansible 구성 관리:**
  - 서버 OS 설정, 보안 하드닝, 패키지 설치 절차의 플레이북(Playbook)화
  - 다중 노드에 대한 동시 설정 배포 및 환경 일관성(Idempotency) 유지

## 2. DB 고가용성 클러스터 구축 (MariaDB Galera)

데이터베이스 계층의 무중단 서비스 및 데이터 동기화 구현

- **Galera Cluster 구축:**
  - 3-Node 기반 동기식 멀티 마스터 복제 클러스터 구성
  - 노드 장애 시 즉각적인 쓰기 가용성 유지 및 데이터 무손실(RPO 0) 보장
- **온프레미스 네트워크 고도화 (MetalLB):**
  - 클라우드 미지원 환경(Bare-metal)에서 `LoadBalancer` 타입 서비스 노출 체계 수립
  - 사설 IP 대역 할당 및 L2 모드 ARP 응답 대행을 통한 외부 접점 확보
- **ProxySQL 부하 분산:**
  - 데이터베이스 전면에 L7 프록시 배치 및 읽기/쓰기 쿼리 자동 분리
  - 노드 상태 실시간 감시를 통한 지능형 장애 조치(Failover) 자동화
- **무중단 배포 (Rolling Update):**
  - **배포 전략:** `order: start-first` 설정을 통해 새 컨테이너 선가동 후 구버전 순차 종료 처리
  - **가용성 제어:** `parallelism: 1` 및 `delay` 설정을 통한 배포 영향도 최소화
  - **자동 롤백 연동:** 헬스체크 실패 시 `failure_action: rollback` 지침을 통한 서비스 안정성 회복 강제화

## 3. K8s 설정 및 배포 자동화 (Helm & GitOps)

선언적 운영 체계 구축을 통한 배포 무결성 및 효율성 극대화

- **Helm Chart 패키징:**
  - K8s 매니페스트의 템플릿화를 통한 재사용성 및 환경별(`dev`/`prod`) 설정 분리 관리
  - 차트 버전 관리를 이용한 애플리케이션 및 인프라 설정의 동시 롤백 체계 수립
- **Argo CD 기반 GitOps 운영:**
  - **신뢰의 단일 원천(Source of Truth):** Git 저장소의 상태를 클러스터에 강제 동기화하여 수동 작업 배제
  - **드리프트 탐지(Drift Detection):** 클러스터 상태 불일치 자동 감지 및 자동 복구(Self-healing) 가동
  - **상세 전략:** [K8s 설정 관리 및 재배포 전략](../../standards/k8s_deployment_strategy.md) 참조

## 4. 유연한 인프라 확장 및 미래 로드맵

부하 변화에 유연하게 대응하는 탄력적 인프라 연구

- **오토 스케일링 (Auto-scaling):**
  - 트래픽 부하 및 리소스 사용량 분석 기반의 VM 인스턴스 동적 확장 체계 연구
  - **지능형 스케일링:** AI 예측 모델 연동을 통한 선제적 자원 증설(Proactive Scaling) 검증
- **하이브리드 확장 및 서비스 메시:**
  - 온프레미스(Proxmox)와 퍼블릭 클라우드 간의 유연한 자원 연동(Cloud Bursting) 기반 마련
  - **Istio 도입:** 사이드카 기반의 정밀 트래픽 제어 및 서비스 간 암호화(mTLS) 통신 연구
