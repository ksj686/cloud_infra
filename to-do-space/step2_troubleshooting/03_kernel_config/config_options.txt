# Key Linux Kernel Configuration Options for Servers

## Networking
- CONFIG_NET_SCHED: For Traffic Control (tc).
- CONFIG_NETFILTER: For ufw/iptables/nftables.
- CONFIG_IP_NF_IPTABLES: Legacy iptables support.
- CONFIG_NF_TABLES: Modern nftables support.
- CONFIG_IPV6: IPv6 support (usually enabled).

## Storage & File Systems
- CONFIG_MD: RAID support (mdadm).
- CONFIG_BLK_DEV_DM: Device Mapper (LVM, dm-crypt).
- CONFIG_EXT4_FS: Standard ext4 filesystem.
- CONFIG_XFS_FS: XFS filesystem (often used in data centers).
- CONFIG_BTRFS_FS: Btrfs filesystem support.

## Security
- CONFIG_SECURITY_APPARMOR: AppArmor support (Ubuntu default).
- CONFIG_SECURITY_SELINUX: SELinux support.
- CONFIG_MODULE_SIG: Module signing (can cause issues with custom drivers if misconfigured).

## CPU & Performance
- CONFIG_NR_CPUS: Maximum number of supported CPUs.
- CONFIG_HZ: Timer frequency (Higher Hz = more responsive, but more context switches).
- CONFIG_PREEMPT: Kernel preemption mode (PREEMPT_VOLUNTARY for servers).

## Critical Build Options (for newer kernels)
- CONFIG_SYSTEM_TRUSTED_KEYS: Often needs to be empty ("") for manual builds.
- CONFIG_SYSTEM_REVOCATION_KEYS: Often needs to be empty ("") for manual builds.
