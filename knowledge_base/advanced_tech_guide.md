# Advanced Infrastructure Technology Guide

본 문서는 프로젝트 고도화를 위해 검토 중인 핵심 기술 스택의 정의, 활용 사례 및 아키텍처적 연관 관계 정리

---

## 1. 고가용성 및 스토리지 (HA & Storage)
- **Ceph (분산 스토리지 시스템):**
    - **정의:** 객체(Object), 블록(Block), 파일(File) 스토리지를 하나의 클러스터에서 제공하는 분산 스토리지 솔루션
    - **활용:** 하드웨어 레벨의 SPoF 제거 및 무중단 용량 확장. Proxmox와 연동하여 VM 디스크의 고가용성 확보
- **Galera Cluster (DB 복제):**
    - **정의:** MariaDB/MySQL을 위한 동기식 멀티 마스터 클러스터링 솔루션
    - **활용:** 모든 노드에 실시간 데이터 복제를 통한 DB 레벨의 무중단 서비스 구현
- **garbd (Galera Arbitrator):**
    - **정의:** 데이터 복제에는 참여하지 않으나 쿼럼(Quorum) 유지에 참여하는 경량 프로세스
    - **활용:** 짝수 노드 구성 시 스플릿 브레인(Split-brain) 현상 방지 및 최소 비용으로 고가용성 유지
- **ProxySQL:**
    - **정의:** MySQL 프로토콜을 이해하는 고성능 DB 레이어 프록시
    - **활용:** Read/Write 분할, 쿼리 캐싱, 장애 노드 자동 제외를 통한 DB 부하 분산

## 2. 모니터링 및 관측성 (Observability)
- **Thanos (멀티 클러스터 프로메테우스):**
    - **정의:** 여러 프로메테우스 서버의 데이터를 통합하여 장기 보관 및 글로벌 뷰를 제공하는 오픈소스
    - **활용:** 멀티 가용 영역(AZ) 또는 하이브리드 클라우드 환경의 중앙 집중 모니터링 구현
- **Loki & Fluent Bit:**
    - **정의:** Fluent Bit(수집기)와 Loki(저장소)를 조합한 클라우드 네이티브 로그 관리 스택
    - **활용:** 하이브리드 환경의 로그를 중앙으로 Push 하여 통합 분석 및 장기 보관
- **k6 (부하 테스트):**
    - **정의:** 개발자 중심의 오픈소스 부하 테스트 도구(JavaScript 기반)
    - **활용:** 시스템 한계 측정 및 p95 Latency(상위 5% 지연 시간) 분석을 통한 인프라 성능 검증

## 3. 네트워크 및 트래픽 제어 (Network & Traffic)
- **pfSense:**
    - **정의:** FreeBSD 기반의 오픈소스 방화벽 및 라우팅 소프트웨어
    - **활용:** 인프라 최상단에서 VPN, IDS/IPS, 트래픽 쉐이핑을 처리하는 강력한 게이트웨이 운용
- **Ingress API:**
    - **정의:** 쿠버네티스 클러스터 외부에서 내부 서비스로의 HTTP/S 경로를 관리하는 API 리소스
    - **활용:** 도메인 기반 라우팅, SSL 종단, 가상 호스팅 통합 관리
- **Cloud Bursting (하이브리드 버스팅):**
    - **정의:** 온프레미스 자원 초과 시 클라우드 자원을 동적으로 확장하여 사용하는 구조
    - **활용:** 비용 효율적인 상시 자원 운용 및 피크 타임 유연성 확보

## 4. 자동화 및 AI (Automation & AI)
- **Argo CD:**
    - **정의:** 쿠버네티스를 위한 선언적 GitOps 지속적 배포(CD) 도구
    - **활용:** Git 저장소의 상태를 클러스터에 자동 동기화하여 수동 작업 배제
- **Karpenter & KEDA:**
    - **정의:** Karpenter(노드 오토스케일러), KEDA(이벤트 기반 오토스케일러)
    - **활용:** 단순히 CPU 사용량이 아닌 RPS(초당 요청 수), 메시지 큐 적체량 기반의 즉각적 자원 확장
- **Predictive AI 모델 (MS AI 예시):**
    - **정의:** 학습된 모델을 통해 인프라 부하를 예측하는 경량 모델
    - **활용:** 과거 데이터를 기반으로 부하 발생 전 선제적 오토스케일링(Proactive Scaling) 수행

## 5. 데이터 가속 및 세션 (Data Acceleration)
- **Redis (Cache & Session):**
    - **정의:** 인메모리 데이터 구조 저장소
    - **활용:**
        - **Cache Redis:** DB 쿼리 부하 감소 및 응답 속도 향상
        - **Session Redis:** 멀티 서버 환경에서 사용자 세션 상태 유지 및 공유
    - **Redis Sentinel:** Redis 노드 감시 및 자동 페일오버를 통한 캐시 레이어 가용성 확보
